# 開発方針

## 目的

nyanta-chatは、訪問診療・在宅医療の初回相談前に必要な情報を、利用者がやさしいチャット形式で入力できるようにするアプリです。

## 環境の役割

```text
MacBook Air
  移動中の軽い編集、仕様整理、GitHub操作、Codexによるコード修正

Ubuntu mini PC
  Docker起動、動作確認、テスト、ビルド確認

VPS
  Docker Composeで本番稼働

GitHub
  履歴管理、ブランチ共有、復元ポイント

NotebookLM
  仕様、議事録、開発履歴、運用メモの参照先
```

## ブランチ運用

- `main`: 本番安定版
- `develop`: 開発統合
- `feature/*`: 機能追加
- `fix/*`: 不具合修正

小さく作り、小さく確認し、小さくコミットします。1つのコミットには1つの意図だけを入れます。

## バージョン運用

- 現在の公開済み機能を `v0.1` とする
- 機能追加ごとに `v0.2`, `v0.3` のようにバージョンを上げる
- 不具合修正だけの場合は必要に応じて `v0.1.1` のように末尾を上げる
- アプリ内表示は `lib/version.ts` の `APP_VERSION` で管理する
- 変更内容は `docs/changelog.md` に記録する

## 作業手順

1. GitHubの最新状態をpullする
2. 機能ごとのブランチを作る
3. コードを変更する
4. UbuntuでDocker起動確認する
5. テスト・lint・ビルドを確認する
6. コミットしてGitHubへpushする
7. 必要に応じて `develop` または `main` へマージする

## コミットメッセージ

```text
feat: 新機能
fix: 不具合修正
docs: ドキュメント変更
chore: 設定・環境整備
refactor: 振る舞いを変えない整理
test: テスト追加・修正
```

## 実装方針

- 既存のNext.js App Router構成を維持する
- 画面部品は `components/` に分離する
- 問診項目は `lib/questions.ts` に集約する
- DB処理は `lib/db.ts` に集約する
- 医療判断や診断はアプリ内で行わない
- Claudeの返答は短いリアクションに限定する

## 確認観点

- 入力が最後まで進むか
- 戻るボタンで1つ前の質問へ戻れるか
- スキップ可能な項目が正しくスキップできるか
- 写真入力が保存・表示できるか
- 完了画面に回答が表示されるか
- `/nyanta` のbasePathで表示できるか
- Docker再起動後もSQLiteデータが保持されるか
