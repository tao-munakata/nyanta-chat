# 開発運用メモ

## 環境の役割

| 環境 | 役割 |
| --- | --- |
| MacBook Air | 移動中の編集、Codex作業、GitHub操作、軽い確認 |
| Ubuntu mini PC | Docker動作確認、テスト、ビルド確認 |
| VPS | Docker Composeで本番またはデモ運用 |
| GitHub | 履歴管理、ブランチ共有 |
| NotebookLM / Notion | 仕様・議事録・開発履歴の整理 |

## ブランチ運用

```text
main        本番安定版
develop     開発統合
feature/*   機能追加
fix/*       不具合修正
```

現在の作業ブランチ:

```text
feature-additions
```

## よく使うコマンド

ローカル開発:

```bash
npm install
npm run dev
```

確認:

```bash
npm test
npm run lint
npm run build
```

Docker:

```bash
docker compose up -d --build
docker compose logs -f
docker compose down
```

Git:

```bash
git status
git log --oneline --decorate -n 20
git push
```

## 注意点

- `.env` はGitHubにpushしない
- `data/` はSQLiteデータなので削除注意
- devサーバー起動中に `npm run build` を実行した後は、`.next` を削除してdevサーバーを再起動すると安全
- MacではNode/npmはユーザー領域の `~/.local` に導入済み
